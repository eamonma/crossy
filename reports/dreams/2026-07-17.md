## Method

Pass 1: ten read-only agents, one per doc cluster, extracted and verified every
checkable claim against code and vectors, returning verdicts with file:line evidence.
Pass 2: one agent walked the merged-PR window for decisions recorded nowhere but the
PR. Pass 3: organization proposals only, below. Repairs follow the direction rule:
descriptive drift fixed in scoped PRs, normative drift filed as `drift/normative`
issues, decision capture written into owning docs with PR provenance.

## Skill amendments adopted this run

Folded into SKILL.md on the #292 branch after this run:

- Issues are grouped per owning doc, itemized and severity-tagged, not one issue per
  drift; a one-owner repo drowns in single-item issues.
- Broken cross-references in normative docs are mechanical repairs and may ride a
  dream PR; the direction rule protects behavior claims, not dead pointers.
- Watermarks stamp only fully-settled docs (see policy above).
- Pass 1 should weight status lines and "later/upcoming/ahead-of" phrasing as the
  highest-risk claims; that is where nearly all of this dream's drift lived.
